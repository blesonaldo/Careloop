CARELOOP BACKEND ARCHITECTURE BREAKDOWN
======================================

OVERVIEW
--------
The Careloop signup form is designed to integrate with a backend system that processes user registration and sends confirmation emails via SendGrid. Here's how the backend architecture works:

FRONTEND TO BACKEND FLOW
------------------------
1. User fills out the signup form (Full Name, Business Name, Email, Phone)
2. Client-side validation ensures all required fields are filled and email format is valid
3. On form submission, the frontend shows loading state for 1.5 seconds
4. Form data is sent to backend API endpoint via POST request
5. Backend processes the data and triggers SendGrid email service
6. Success response triggers the success popup overlay

BACKEND COMPONENTS
------------------

1. API ENDPOINT (Node.js/Express)
   - POST /api/auth/signup
   - Receives JSON payload: { fullName, businessName, email, phone, dialCode }
   - Validates input data server-side
   - Checks for duplicate email addresses
   - Creates user record in database
   - Generates verification token
   - Triggers SendGrid email service
   - Returns success response

2. DATABASE SCHEMA (MongoDB/PostgreSQL)
   Users Collection:
   - _id: ObjectId
   - fullName: String
   - businessName: String  
   - email: String (unique)
   - phone: String
   - dialCode: String
   - createdAt: Date
   - verificationToken: String
   - isVerified: Boolean
   - status: String ('pending', 'active', 'suspended')

3. SENDGRID INTEGRATION
   - SendGrid API Key stored in environment variables
   - Email template ID for signup confirmation
   - Dynamic template data: fullName, businessName, verificationLink
   - Email content includes:
     * Welcome message
     * Account details
     * Verification link
     * Support contact information

4. AUTHENTICATION & SECURITY
   - JWT tokens for session management
   - Password hashing with bcrypt
   - Rate limiting on signup endpoint
   - Input sanitization and validation
   - CORS configuration for frontend domain

5. ERROR HANDLING
   - Duplicate email detection
   - Invalid phone number format
   - SendGrid service errors
   - Database connection issues
   - Proper HTTP status codes and error messages

TECHNOLOGY STACK
-----------------
- Backend: Node.js with Express.js
- Database: MongoDB with Mongoose ODM
- Email Service: SendGrid API
- Authentication: JWT + bcrypt
- Validation: Joi or express-validator
- Environment: dotenv for configuration

API ENDPOINTS
-------------
POST /api/auth/signup
{
  "fullName": "John Doe",
  "businessName": "Doe Enterprises", 
  "email": "john@example.com",
  "phone": "5551234567",
  "dialCode": "+1"
}

Response:
{
  "success": true,
  "message": "User registered successfully",
  "userId": "64a7b8c9d1e2f3g4h5i6j7k8"
}

SENDGRID EMAIL TEMPLATE
------------------------
Subject: Welcome to Careloop - Verify Your Account

Content:
- Personalized greeting with fullName
- Business name confirmation
- Verification button/link
- Expiration notice (24 hours)
- Support contact information
